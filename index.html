BSC Global · Simulation Consultant & Analytics Engineer · Mining Sector
2025 – 2026

End-to-end simulation analytics ecosystem evaluating a high-volume outbound logistics chain across rail, dual-port, and vessel systems — built on a 48-node discrete event simulation model. Sole analytics engineer responsible for database architecture, automation workflows, cost modelling engine, AI agent deployment, and client-facing sprint delivery.

  • Automated Simulation Workflow Engine: Parameterised Excel-driven scenario creation, atomic bulk-insert transactions, idempotent OAT sensitivity generation (±20% across 10 parameters), and FK-ordered cascade deletion with rollback-on-error — reducing scenario setup from hours to minutes.
  • Scalable Output Ingestion Pipeline: Processed 1,000–9,000 simulation CSV files via parallel ThreadPoolExecutor reads, SQL-side deduplication via staging tables, and fast_executemany batched inserts with pre-flight sync and full re-run safety.
  • Enterprise Simulation Database Architecture: 48-node Azure SQL schema with 13 dimension/fact tables, 7 Power BI views, warmup-excluded pre-aggregated summary tables, and a pre-computed OAT tornado sensitivity view — enabling sub-second KPI queries across stochastic replications.
  • Production-Grade Cost Modelling Engine: Modular Python package with CLI covering 4 cost categories, event-based demurrage calculation, port fixed-cost scaling, replication-level aggregation, and multi-sheet Excel report generation.
  • AI Analytics Agent Design & Deployment: Two principal-level AI agents with anti-hallucination contracts, dynamic procedural memory (39 contexts / 9 categories), schema-grounded SQL generation, and role-specific specialisations.
  • Client Analytics Delivery Framework: 4-sprint structured analysis (Monte Carlo, sensitivity, cost model) with Power BI dashboards, tornado charts, and executive sprint presentations informing port allocation, buffer sizing, and fleet decisions.
